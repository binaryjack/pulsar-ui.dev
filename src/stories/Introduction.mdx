# Atomos Prime Design System

Welcome to **Atomos Prime** - A Pulsar-based design system with TailwindCSS.

## Architecture

This component library follows **Atomic Design principles**:

### Atoms
Single-purpose UI elements (Button, Input, Typography, etc.)
- ✅ External constants for default config/styling
- ✅ No inline logic in components
- ✅ ComponentConfigBuilder for configuration
- ✅ ComponentStylingBuilder for styling
- ✅ Ternary operators in return statements
- ✅ Skeleton component for loading states

### Molecules
Compositions of 2-3 atoms (Badge, Label, RadioGroup, etc.)
- Compose atomic components
- Maintain separation of concerns
- Follow same architectural patterns as atoms

### Organisms
Complex compositions (Card, Header, Footer, Select, etc.)
- Compose molecules and atoms
- Handle complex interactions
- Maintain architectural consistency

## Key Patterns

### 1. External Constants
```typescript
const defaultConfig = new ComponentConfigBuilder('primary').build()
const defaultStyling = new ComponentStylingBuilder().base('...').build()
```

### 2. Ternary in Return
```typescript
export const Component = (props) =>
  props.loading ?
    <Skeleton /> :
    <Element {...props} />
```

### 3. No Inline Logic
```typescript
// ❌ Bad
className={`base ${loading ? 'loading' : ''}`}

// ✅ Good
className={cn(
  styling.base,
  loading ? styling.loading : '',
  config.className
)}
```

### 4. Type Safety
```typescript
export interface IComponentProps extends Pulsar.HtmlExtends<'element'> {
  config?: IComponentConfig
  styling?: IComponentStyling
}
```
